SCR-00 Map dashboard — hero screen All users
Top navbar
GullyFix logo left — Syne 800, cream on brown
Ward selector pill — "Ward 3 · Gandhipuram" dropdown, amber border
XP chip right — "⭐ 450 XP" amber badge, links to SCR-08
Profile avatar — initials circle, links to SCR-08
Stat strip — below nav, full width
4 stat chips in a row: "47 resolved" · "6.2 avg days" · "68 ward health" · "128 total open" — cream-dark bg, brown pixel border
Map controls bar
Layer toggles — 3 pill buttons: "Live issues" (default on) · "Heatmap" · "AI predictions" — pixel border style
Category filter — icon buttons for Road / Water / Waste / Light / All
Map canvas (Leaflet + OpenStreetMap)
Issue pins — square pixel pins, emoji icon + category color (terracotta/sky/sage/amber). Hard brown border.
Heatmap layer — Leaflet.heat, amber/terracotta color ramp. Toggle shows density clusters.
Prediction pins — sky-blue square with pulsing CSS border animation. Shows on toggle.
Pin click → opens bottom sheet (SCR-07 condensed) without leaving map
FAB — floating action button
"+ Report issue" — terracotta, bottom-right, Syne font, pixel shadow. Tapping goes to SCR-02.

SCR-02 → 04 Report flow — 3-step wizard Citizen
Step indicator (all 3 screens)
3 square step nodes connected by lines — active = terracotta fill, done = sage fill, upcoming = cream. Pixel border style.
SCR-02 — Photo capture
Upload zone — large dashed pixel border square, "📸 Take a photo or upload" — tapping opens camera/file
Preview — once selected, image fills the zone with a sage "✓ Photo ready" badge overlaid
Optional description — textarea, max 500 chars, "Describe what you see (optional)"
"Next →" button — terracotta primary, disabled until photo selected
SCR-03 — Location
Mini Leaflet map — shows user's GPS pin, draggable if permission denied
Address display — reverse geocoded address below map in pixel card
"Use my location" button — secondary style, triggers geolocation API
"Next →" button — disabled until location confirmed
SCR-04 — Review and submit
Photo thumbnail — left column, 120px square
Location summary — address + ward name
XP preview chip — "You'll earn +20 XP for this report" amber badge
"Submit report" button — terracotta primary, full width. Triggers Firestore write + Cloud Function.
SCR-05 AI agent trace — most important screen Citizen
Purpose — this screen wins agentic depth scoring
Shows the 4-step triage pipeline running in real time. Polls Firestore issue doc. Each step appears sequentially as the Cloud Function progresses. Judges watch AI thinking live.
Header
"AI is analysing your report" — Syne 700 heading. Amber pulsing dot animation next to it while running.
Step checklist — 4 rows, animate in sequentially
Step 1 — "Reading your photo" · Cloud Vision · shows top 3 labels once done e.g. "road · asphalt · damage"
Step 2 — "Classifying issue" · Gemini · shows category + priority score + department once done
Step 3 — "Checking nearby reports" · Firestore geo-query · shows "No duplicates found" or "Merged with Issue #42"
Step 4 — "Routing to department" · shows "Roads and Infrastructure Dept — notified"
Result card — appears after all 4 steps
Category badge · Priority score bar · AI reasoning text (aiReasoning field) · Resolution steps (3 bullets, Gemini-generated) · Department routed to
CTA
"+20 XP earned!" amber badge animates in · "View on map" secondary button · "Report another" ghost button

Priority bar — labelled 0–100 with colour fill (red/amber/sage by range)
Status badge — Open / In progress / Resolved with colour coding
Verify count — "👥 7 people have seen this" with the verify CTA button
AI advisory panel — sky-light pixel card
Header — "AI recommendation for Roads dept" with Gemini logo mark
3 resolution steps — bulleted, from resolutionSteps[] field. This is where judges see Gemini advising.
AI confidence score — "92% confidence" small badge
Citizen actions
"I've seen this too" — primary btn, triggers verifyIssue callable. Disabled if already verified or within cooldown.
"Flag as invalid" — ghost/danger btn, triggers flagIssue callable

SCR-11 Admin issue ledger Admin only
Filter bar
3 dropdowns — Ward · Category · Status — pixel select style. Resets Firestore query on change.
Sort — Priority (default) · Date · Verify count
Issue table rows
Each row: photo thumb · category badge · title · ward · priority bar · verify count · status badge · "Update status" dropdown
Status dropdown inline — Open → In progress → Resolved. On change calls updateIssueStatus callable. Row turns sage on resolve.
Row tap → SCR-07 issue detail with full AI advisory visible
Pending review queue
Flagged issues (flagCount ≥ 3) shown in a separate red-light section at top — "3 issues need review"
